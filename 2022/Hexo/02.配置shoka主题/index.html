# 2. 评论管理及提醒功能

Valine-Admin-Github官网

前往Valine-Admin-Github官网

  • 进入 Valine-Admin-Github 官网，点击 Fork：
  • 打开 LeanCloud 网站，登录，进入你刚刚创建 hexo 评论应用里；

第一步：部署

  • 点击 云引擎 → WEB → 部署 → 部署项目，点击 Git 部署，配置 Git;
  • 将刚刚 Fork 之后的网址，复制粘贴进去（即：https://github.com/ 你的 GitHub 名 / Valine-Admin），点保存；
  • 然后再点进去部署项目，Git 部署，进入如下页面，点击部署：

第二步：域名绑定

  • 点击 设置 → 域名绑定 → 绑定新域名，输入域名，点击绑定；
  • 点击 云引擎 → WEB →设置，设置中访问域名中的共享域名添加你绑定的域名后缀前面的名字；

第三步：自定义环境变量

Akismet官网

前往Akismet官网地址

  • 在云引擎 → WEB → 设置，设置中自定义环境变量 → 添加，添加以下环境变量：

    彩色留言评论通知（只有博主的通知没有彩色效果）：
  • MAIL_TEMPLATE 的值：
  • MAIL_TEMPLATE_ADMIN 的值:

第四步：解决 LeanCloud 流控问题

  • 首先，进入 GitHub 登录，鼠标放在右上角，点击 setting → Developing settings → Personal access tokens, 添加一个新的 TOKEN;

设置名字为 GITHUB_TOKEN , 然后勾选 repo , admin:repo_hook , workflow 等选项，最后点击 Generate token 。
这个 TOKEN 主要用来启动 actions 和上传结果用的；

TOKEN 名字必须是 GITHUB_TOKEN!

  • 在 Github 中搜索 WakeLeanCloud，点击进入其中一个仓库（任何一个都行），点击 Fork;
  • 成功 Fork 之后，进入项目的 Settings，点击 Secrets → Actions → New repository secret, 添加你的评论后台地址

Name 的名字必须是 SITE，Value 可以是多个地址，用英文逗号分隔。注意：不能使用中文逗号


评论后台地址指 云引擎 WEB 中部署 里面生产环境下面的链接
评论管理注册地址： https://xxxx.avosapps.us/sign-up
评论管理地址 (后台地址)： https://xxxx.avosapps.us/

  • 然后，对自己的项目点个 star 就能启动了，启动后请切换到 actions，看看是否运行成功。
    自己点自己的项目是手动执行一次 actions, 并不是不点 star 这个 actions 就不会运行。
