# 编译警告 MSB8028

中间目录 (shared-intermediate-path) 包含从另一个项目 (intermediate-path) 共享的文件 。 这可能导致错误的清理和重新生成行为。

VS 中，一个解决方案中有多个子项目时，出现 MSB8028 这个显示警告。

# 解决方法

  • 打开这个解决方案文件夹
  • 找到正在启用的子项目的项目文件夹并打开
  • 打开此项目文件夹，手动删除 "Debug" 文件夹 (即：解决方案 \ 子项目 \Debug)
  • 再重新生成 Debug 文件夹。
